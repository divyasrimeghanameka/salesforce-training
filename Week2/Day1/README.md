# Day 1 - Lightning Web Components (LWC)

## What is LWC?
Lightning Web Components (LWC) is Salesforce’s modern framework for building dynamic and reusable user interface components for enterprise applications.

LWC is built using:

- HTML  
- JavaScript  
- CSS  
- Metadata XML  

It enables developers to design responsive and scalable applications.

---

## Why Salesforce Uses LWC
Salesforce adopted LWC because:

- It delivers better speed and efficiency  
- Uses latest web technology standards  
- Encourages reusable UI architecture  
- Enhances application performance  
- Simplifies frontend development  

LWC gradually replaced Aura Components due to improved maintainability and faster rendering.

# Frontend vs Backend Thinking

## Frontend / UI Logic

Handled in Frontend:

- Button interactions  
- User form inputs  
- Notification visibility  
- Navigation between pages  

Frontend mainly focuses on user experience and interface display.

---

## Backend / Apex Logic

Handled in Backend:

- Database transactions  
- Fee and attendance calculations  
- Advanced validations  
- Business process implementation  
- Record creation and updates  

Backend mainly manages data operations and business rules.

---

# Reflection
Modern enterprise applications use component-oriented architecture because:

- Applications become more organized  
- Components can be reused across screens  
- Maintenance becomes simpler  
- Teams can develop independently  
- UI consistency improves across the platform  

Component-based development improves scalability and productivity.

# Revision Questions

## 1. What is a component?
A component is an independent and reusable section of a user interface.

---

## 2. Why are reusable components useful?
They help reduce redundancy and simplify maintenance.

---

## 3. Difference between frontend and backend?
Frontend manages interface and user interaction.  
Backend handles logic, processing, and database operations.

---

## 4. Why did Salesforce move toward LWC?
To achieve higher performance and support modern web development standards.

---

## 5. Why is UI important in enterprise systems?
A well-designed UI improves usability and user efficiency.

---

## 6. Why should systems separate UI and business logic?
It improves security, scalability, and maintainability.

---

## 7. What security risks exist in enterprise applications?
Unauthorized access, insecure permissions, and data exposure.

---

## 8. Why should developers think modularly?
Modular systems are easier to reuse, manage, and extend.
